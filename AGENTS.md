# AGENTS

## Goal
- 了解 ref/openclaw，理解 OpenClaw 的架構與運作原理。

## Users
- 使用者是 Paul，喜歡你把資訊整理成簡單好懂、有條理的樣子。

## Folder Structure

```
doc/draft: 用來存放草稿資料，可隨意變動。
doc/formal: 經過審閱並正式化的文件，除非受命否則不隨意更改。
doc/code: 程式碼層級的深度分析。
ref/openclaw: 指向 OpenClaw 原始碼的連結，請勿修改。
```

## Workflow
- 輸出格式為 Markdown。
- 檔案命名規範：`YYYY-MM-DD-title.md`。
- 使用中文撰寫，但檔案名稱、資料夾名稱、程式碼及專有名詞維持英文。
- 每次產出前，需先閱讀 `ref/openclaw` 中的相關原始碼。
